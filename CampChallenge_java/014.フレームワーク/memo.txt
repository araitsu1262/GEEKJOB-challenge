Spring Framework

概要
オープンソースフレームワークで、単にSpringと呼ばれることがある。
データベースへのアクセスやトランザクション処理などをサポートするフレームワークをはじめ、多数のフレームワークの集合体。

特徴

・DI(Dependency Injection:依存性注入)

オブジェクトが他のオブジェクトを利用するコードを「依存性」と呼び、実行時に結合する設計思想のこと。
DIはプログラム同士の依存性を排除し、結果的にコードの簡素化、テストの簡易化、コード改修・移行の簡易化が見込められる。

・AOP(Aspect Oriented Programming:アスペクト指向プログラミング)

属性(データ)と操作(メソッド)の集合であるクラスに分解できない要素を「アスペクト」としてモジュール化し、管理・変更・追加を容易にすること。
元のソースコードに変更を加えずに、新たな処理を追加することができる。


Java EE(Java Platform,Enterprise Edition)

概要

Java標準仕様のフレームワーク。
EJBやJavaの各種APIなど、大吉備システムの構築に必要な機能がまとめて提供されている。

特徴

・フルスタックフレームワーク

ServletやJSP、EJPやJavaの各種APIなど、大規模システムの構築に必要な機能がまとめて提供されている。

・標準=Oracle社のサポートあり

世界中の技術者がチェックしているフレームワークでもあり、参考になる記事や書籍もある。
ただ大きなフレームワークだけに更新頻度は遅く、最新技術を反映させるにも標準仕様としての安定感が重視されている。


Play Framework

概要

JavaとJavaとの連携が容易なプログラミング言語Scalaで書かれたオープンソースのフレームワーク。

特徴

軽量さと生産性が高い。
CPUリソースやメモリ使用量が少ないので、軽快に動作する。
Java標準のWebアプリケーション仕様であるJava EEに準拠していない代わりに、シンプルに開発が進められるメリットがある。
